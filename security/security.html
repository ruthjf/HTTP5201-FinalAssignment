Security

Security is inspected through Chrome DevTools. The site is secure and using valid HTTPS. TLS certificate used is valid through the Netlify app. The connection to the site is encrypted. https://securityheaders.com/ was used to determine the best HTTP Headers configuration. No sensitive data is displayed or exposed.

HTTP Headers Configuration:

Content Security Policy : prevent XSS attacks through whitelisting approved content sources

Referrer Policy: allows a site to control navigation information in the browser

HTTP Strict Transport Security : strengthens TLS implementation

X-Frame Options: allows the site to be framed or not, defends from attacks like clickjacking

X-XSS-Protection: configuration for the built in XSS Auditor
